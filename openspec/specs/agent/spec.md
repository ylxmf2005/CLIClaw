## Overview

An agent is a named principal that can authenticate to the daemon via a token and process envelopes addressed to `agent:<name>`.

## Identity

### Name

Agent names must match `^[A-Za-z0-9]+(?:-[A-Za-z0-9]+)*$`:
- alphanumeric characters, hyphens allowed (not at start/end; not consecutive)
- uniqueness checked case-insensitively

### Token

Generated by `src/agent/auth.ts`: 32 random hex characters. Printed once by `hiboss setup` and `hiboss agent register`. Stored as plaintext in SQLite.

## Authorization

The daemon authorizes token-authenticated operations using `config.permission_policy`.

- Each agent has a `permissionLevel`: `restricted`, `standard`, `privileged`, or `admin`.
- `permissionLevel: admin` grants authorization only; channel `fromBoss` / `[boss]` are derived from token-based channel login + token policy role.

## Providers

| Provider | CLI Command | Default Home |
|----------|-------------|--------------|
| claude | `claude` (Claude Code CLI) | `~/.claude` |
| codex | `codex` (Codex CLI) | `~/.codex` |

### Provider Configuration

| Setting | Description | Values |
|---------|-------------|--------|
| `provider` | SDK provider | `claude`, `codex` |
| `model` | Model override | Provider-specific model ID (`null` = provider default) |
| `reasoningEffort` | Reasoning effort override | `none`, `low`, `medium`, `high`, `xhigh` (`null` = provider default) |
| `metadata.providerCli.<provider>.env` | Per-agent env overrides | String env map |
| `permissionLevel` | Authorization for CLI/RPC ops | `restricted`, `standard`, `privileged`, `admin` |
| `sessionPolicy` | Session refresh policy | See `openspec/specs/agent/sessions.md` |
| `relayMode` | Default interactive relay behavior for new chats | `default-on`, `default-off` |

### Provider Homes

By default, provider state is shared across all agents using default homes (`~/.claude`, `~/.codex`).

At runtime, Hi-Boss first clears provider-home override env vars (`CLAUDE_CONFIG_DIR`, `CODEX_HOME`), then applies per-agent `metadata.providerCli.<provider>.env` overrides if configured.

During agent-home setup, Hi-Boss ensures `~/.codex/config.toml` has `[features] multi_agent = true`.

### Provider Execution Mode

Hi-Boss runs provider CLIs in **full-access mode**:
- Codex: `--dangerously-bypass-approvals-and-sandbox`
- Claude: `--permission-mode bypassPermissions`

### Invocation (canonical)

Claude (per turn):
- `claude -p --append-system-prompt ... --output-format stream-json --verbose --permission-mode bypassPermissions`
- Adds `--add-dir` for `{{HIBOSS_DIR}}/agents/<agent>/internal_space`
- Adds `--model <model>` when configured, `-r <session-id>` when resuming
- Reasoning effort via env: `none`/`low` → `CLAUDE_CODE_EFFORT_LEVEL=low`, `medium` → `medium`, `high`/`xhigh` → `high`

Codex (per turn):
- Fresh: `codex exec --json --skip-git-repo-check --dangerously-bypass-approvals-and-sandbox --add-dir ... -c developer_instructions=... [-m <model>] <prompt>`
- Resume: `codex exec resume --json --skip-git-repo-check --dangerously-bypass-approvals-and-sandbox [-m <model>] <thread-id> <prompt>`
  - Note: `codex exec resume` does not support `--add-dir`.

One-shot execution (`metadata.oneshotType = isolated|clone`):
- Routed by daemon to `OneShotExecutor` (`src/agent/oneshot-executor.ts`) instead of the main session queue.
- Uses the same provider turn runners as normal agent execution (system instructions + agent token included).
- `clone` attempts to clone current provider session context; falls back to `isolated` if no clone source is available.

### Token Usage

| Metric | Meaning | Logged | Persisted |
|---|---|---|---|
| `context-length` | Final model-call size (prompt + output) | Always | `agent_runs.context_length` |
| `input-tokens` | Total input tokens (billing) | Debug only | No |
| `output-tokens` | Total output tokens (billing) | Debug only | No |
| `cache-read-tokens` | Cache hits | Debug only | No |
| `cache-write-tokens` | Cache writes | Debug only | No |
| `total-tokens` | `input + output` | Debug only | No |

Claude `context-length` = `input_tokens + cache_read_input_tokens + cache_creation_input_tokens + output_tokens`.

Codex `context-length` from rollout log: `last_token_usage.input_tokens + last_token_usage.output_tokens`.

## Instruction Generation

System instructions are regenerated per new session and passed inline to the CLI. On each new session, Hi-Boss injects:
- truncated `MEMORY.md`, recent daily memory files, session summary snapshots from history markdown files.

Template system: Nunjucks (`src/shared/prompt-renderer.ts`, `src/shared/prompt-context.ts`). Entry: `prompts/system/base.md`.

## Agent Execution

### Execution Flow

1. **Trigger**: New/scheduled/startup envelope
2. **Lock**: Per-agent queue lock (no concurrent runs for same agent)
3. **Session**: Get or create (see `openspec/specs/agent/sessions.md`)
4. **Turn Input**: Format pending envelopes
5. **Execute**: Spawn provider CLI
6. **Auto-Ack**: Mark read envelopes `done` (at-most-once)
7. **Audit**: Record in `agent_runs`
8. **Reschedule**: If more pending, schedule another turn

`MAX_ENVELOPES_PER_TURN = 10`

### Abort / Cancellation

Provider CLIs can be aborted by terminating the child process (SIGINT/SIGTERM).

## One-shot Executor

One-shot envelopes (`metadata.oneshotType` = `isolated` or `clone`) run as ephemeral jobs.

- Trigger sources: Telegram `/isolated` and `/clone`, plus cron `execution-mode` `isolated|clone`.
- Concurrency: default max `4` (clamped `1..32`).
- At-most-once: request envelope is ACKed immediately, then executed asynchronously.
- Response routing: replies go to the original envelope sender (or `metadata.cronResponseTo` for one-shot cron materializations).
- Session isolation: one-shot runs do not switch or pollute the main chat session (`chat-session-changed: false` in one-shot responses).
